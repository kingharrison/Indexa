MCP & API Integration

Plug your knowledge base into
any AI tool.

Indexa ships with a built-in MCP (Model Context Protocol) server and a fully OpenAI-compatible API. Connect Claude Desktop, LM Studio, Cursor, or any OpenAI client directly to your local knowledge bases.

🤖

MCP Server — Claude Desktop & Cursor

SSE + HTTP transport with three tools: query, search, and list_collections. Works with stdio clients via mcp-remote bridge.

⚡

OpenAI-Compatible API

Drop-in replacement for OpenAI clients. Collections appear as models. Supports /v1/chat/completions and /v1/models — no code changes needed.

🔍

Hybrid Search

BM25 keyword search + semantic vector search, merged via Reciprocal Rank Fusion. Optional query decomposition and LLM reranking for complex questions.

💬

Multi-Turn Conversations

Full conversation history with context. Cross-collection search lets you query all your knowledge bases at once in a single question.

client = OpenAI(
  base_url="http://localhost:11435/v1",
  api_key="your-api-key"
)

response = client.chat.completions.create(
  model="Research Notes", # your collection

Features

Everything included.
Nothing held back.

📄

15+ File Formats

PDF, DOCX, TXT, Markdown, RTF, HTML, XLSX, PPTX, JSON, XML, YAML, CSV, log files, and source code. Drag and drop directly into the app.

🌐

Web Scraping & Crawling

Paste a URL to index a single page or crawl an entire site with configurable depth and page limits. SHA-256 change detection keeps content fresh automatically.

🔍

Hybrid Search

BM25 keyword + semantic vector search merged via Reciprocal Rank Fusion. Optional query decomposition and LLM reranking for complex, multi-part questions.

New in v1
🗂

Collections

Organize documents into named collections with custom system prompts, refresh intervals, and query context. Cross-collection search available.

🔒

Collection Protection

Password-protect in two modes: Sources Masked — answers only, no citations — or Read Only — view and query, no modifications. PBKDF2 with timing-safe comparison.

✨

AI Distillation

AI rewrites documents for better retrieval accuracy while preserving all facts, names, and technical details. Checkpoint recovery means long distillations survive crashes.

📦

AES-256 Encrypted Exports

Export as portable .indexa bundles with AES-256-GCM encryption and PBKDF2 key derivation. Recipients query your knowledge — never your sources.

🤝

Multi-Provider Support

Ollama by default, or any OpenAI-compatible server. Use separate providers for chat and embeddings. Per-provider API keys and connection testing.

🔄

Auto-Refresh

Web sources re-index on a schedule — hourly, daily, or weekly. SHA-256 content hashing means only changed content triggers a re-ingest.

REST API

A complete API.
OpenAI-compatible.

Indexa runs a local HTTP server on port 11435. It speaks native REST and OpenAI-compatible endpoints — so existing tooling just works, no code changes required.

API key authentication, CORS restricted to localhost, and rate limiting on password attempts (10 tries, 5-minute lockout). Protection-aware — masked collections strip sources from all API responses automatically.

Under the Hood

Built native.
Zero external dependencies.

Platform
macOS · Native SwiftUI + AppKit · Swift 6
Concurrency
Actor-based (KnowledgeEngine, OllamaService, HTTPServer, WebScraper)
Database
SQLite3 (raw C API) + FTS5 for hybrid keyword search
Search
Cosine similarity + BM25 via Reciprocal Rank Fusion
Embeddings
nomic-embed-text via Ollama (768-dim) · batch processing
Encryption
AES-256-GCM (CryptoKit) + PBKDF2 (CommonCrypto, 100k iterations)
HTTP Server
Apple Network framework (NWListener) · port 11435
MCP Transport
SSE + HTTP POST · 30-second keepalive
Compression
LZFSE (Apple built-in) + Gzip for bundles
Sandbox
App Sandbox · network client + server entitlements
